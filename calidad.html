Gestión de la Calidad del Software

Inicio Gestión de la Calidad SDLC Metodologías Ágiles

La gestión de la calidad del software es fundamental para asegurar que los productos desarrollados cumplan con los requisitos y expectativas del cliente, optimizando recursos y reduciendo el riesgo de errores.

Dimensiones clave de calidad (según ISO 9126/25000)

• Funcionalidad: El software cumple con las funciones para las que fue diseñado.

• Fiabilidad: La capacidad del software para realizar su función de manera consistente.

• Usabilidad: La facilidad con la que los usuarios pueden interactuar con el software.

• Eficiencia: El software usa los recursos de manera óptima.

• Mantenibilidad: Qué tan fácil es modificar el software para corregir errores o adaptarlo a nuevos requerimientos.

• Portabilidad: La facilidad con la que el software puede ser adaptado a diferentes entornos.

Las pruebas de software son esenciales para garantizar la calidad del producto. Se clasifican en tipos, niveles y técnicas específicas.

Tipos de Pruebas

Pruebas Funcionales

Evalúan qué hace el sistema, según los requisitos funcionales.

• Caja Negra: Se prueban las funcionalidades sin conocer el código interno. Se enfoca en las entradas y salidas del sistema.

• Aceptación: Pruebas realizadas por el cliente o usuario final para verificar si el sistema cumple con lo acordado (requisitos del negocio).

• Regresión: Se asegura que las funcionalidades que ya funcionaban siguen haciéndolo después de cambios, correcciones o nuevas funcionalidades.

• Integración (funcional): Prueban la interacción entre dos o más módulos para verificar que trabajen correctamente juntos.

Pruebas No Funcionales

Evalúan cómo lo hace el sistema, no qué hace.

• Rendimiento (Performance): Evalúa la rapidez, capacidad de respuesta y estabilidad del sistema bajo ciertas condiciones de carga.

• Seguridad: Evalúa si el sistema protege los datos y mantiene la confidencialidad, integridad y disponibilidad.

• Usabilidad: Verifica qué tan fácil e intuitivo es para los usuarios interactuar con el sistema.

• Compatibilidad: Evalúa si el sistema funciona correctamente en diferentes entornos (navegadores, dispositivos, sistemas operativos, resoluciones, etc.).

Niveles de Pruebas

Refieren a cuándo y sobre qué parte del software se hacen las pruebas.

• Pruebas Unitarias: Se prueban unidades pequeñas e individuales de código, como funciones o métodos. Normalmente las realiza el desarrollador.

• Pruebas de Integración: Se verifica que múltiples módulos o componentes funcionen correctamente cuando se integran entre sí.

• Pruebas de Sistema: Se prueba el sistema completo, como un todo, para asegurarse de que cumple con los requisitos funcionales y no funcionales.

• Pruebas de Aceptación: Se realizan antes de liberar el producto. Las hace el cliente o usuario para validar que el software está listo para producción.

Técnicas de Pruebas

Caja Negra

Pruebas sin conocimiento del código interno. Se enfoca en entradas y salidas.

• Partición de Equivalencia: Divide las entradas en grupos donde se espera el mismo comportamiento para reducir la cantidad de casos de prueba.

• Análisis de Valores Límite: Se prueban los valores en los extremos del rango permitido, donde suelen aparecer errores (por ejemplo, si una edad válida es de 18 a 60, se prueban 17, 18, 60, 61).

Caja Blanca

Pruebas con acceso al código. Se enfoca en la lógica interna.

• Cobertura de Código: Mide cuánto del código fue ejecutado durante las pruebas (líneas, ramas, condiciones).

• Análisis de Caminos: Analiza todos los caminos posibles que puede tomar la ejecución del código para cubrir todos los flujos lógicos.

Caja Gris

Pruebas con conocimiento parcial del código. Mezcla técnicas de caja blanca y negra.

• Pruebas de Interfaz: Verifican la correcta comunicación entre distintos módulos o entre el sistema y componentes externos.

• Pruebas de API: Evalúan si las interfaces de programación (REST, SOAP, etc.) funcionan como se espera al enviar y recibir datos.

Las herramientas de gestión de pruebas, seguimiento de defectos y análisis de calidad son componentes esenciales en la gestión de la calidad del software. Proveen un marco estructurado para asegurar que el software cumpla con los requisitos y expectativas del cliente, al tiempo que optimizan los recursos y reducen el riesgo de errores. Su uso adecuado no solo mejora la calidad del producto, sino que también contribuye a un proceso de desarrollo más ágil y eficiente.

Selenium

Selenium: Conjunto de herramientas que permite simular la interacción de un usuario con un navegador de forma automática y controlada por código.

• Selenium WebDriver: Automatiza el navegador con código.

• Selenium IDE: Herramienta visual (extensión del navegador) para grabar acciones y generar código.

• Selenium Grid: Permite ejecutar pruebas en paralelo en múltiples máquinas y navegadores.

Trello

Trello: Herramienta de gestión de proyectos y organización de tareas basada en Kanban.

Funciones:

• Organizar tareas personales o de equipo

• Hacer listas de tareas pendientes

• Controlar el progreso de un proyecto (pendiente, en proceso, completado)

• Recordatorios, metas, ideas

• Gestionar proyectos

• Asignar tareas a miembros del equipo

• Establecer fechas límite (deadlines)

• Añadir etiquetas, checklists, comentarios, archivos

Colaboración en equipo:

• Varias personas pueden trabajar en el mismo tablero en tiempo real

• Ideal para equipos remotos o distribuidos

Planificación:

• Proyectos de desarrollo de software

• Planificación de eventos

• Calendarios editoriales

• Planes de estudio

Jira

Jira: Herramienta de gestión de proyectos, desarrollada por Atlassian, que se utiliza principalmente para el seguimiento de errores (bugs), la planificación de tareas y la gestión del desarrollo de software ágil (Scrum, Kanban, etc.).

Usos principales:

• Gestión de proyectos de software

• Planificar versiones (releases)

• Dividir el trabajo en tareas o historias de usuario

• Asignar tareas a miembros del equipo

Metodologías ágiles:

• Scrum: Sprints, historias de usuario, backlog, burndown charts

• Kanban: Flujos de trabajo visuales y flexibles

Seguimiento de errores (bug tracking):

• Reportar, asignar y resolver errores en el código

• Registrar comentarios, adjuntar capturas o archivos

Gestión del flujo de trabajo (workflow):

• Personalizar el ciclo de vida de una tarea (Por hacer → En desarrollo → En revisión → Hecho)

Colaboración en equipo:

• Comentarios en tareas

• Notificaciones y menciones

Integración con otras herramientas:

• GitHub, Bitbucket, Confluence, Slack
